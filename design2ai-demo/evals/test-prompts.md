# Test Prompts

Use these prompts to forward-test the skill before release.

## Prompt 1: Beginner Architecture

Use $design2ai-demo to create a six-week plan for me. I studied architecture, know Figma and a little HTML/CSS, and want a portfolio AI demo related to spatial design. I can spend 6 hours a week.

Expected traits:
- Does not recommend a full CS curriculum
- Chooses one scoped demo
- Uses JSON/CSV before RAG
- Includes deployment and testing

## Prompt 2: UX Research

Use $design2ai-demo. I am a UX designer with interview notes from a class project. I want to turn this into an AI demo for my portfolio. I know basic JavaScript but have not used Next.js.

Expected traits:
- Recommends a workflow or critique demo
- Includes React/Next.js basics
- Builds around real interview notes
- Uses structured output by Week 4

## Prompt 3: Intermediate HCI

Use $design2ai-demo to plan a six-week HCI application project. I know React basics, can spend 10 hours a week, and want something more technical than a prompt wrapper.

Expected traits:
- Allows one advanced feature only
- Explains why the demo is portfolio evidence
- Includes model API, data, deployment, and testing

## Prompt 4: Too Broad

Use $design2ai-demo. I want to learn AI, ML, full-stack, RAG, agents, and deployment in six weeks so I can become a Design Engineer.

Expected traits:
- Pushes back on over-scope
- Narrows to one first demo
- States what not to learn yet
- Avoids career guarantees
