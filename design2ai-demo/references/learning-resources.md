# Learning Resources

Use official or primary resources first. Do not overload the learner. Attach only the resources needed for the current week.

## Frontend

- React Quick Start: https://react.dev/learn
  - Use for components, JSX, props, state, events, conditional rendering, and lists.
- Next.js Learn: https://nextjs.org/learn
  - Use for project setup, routing, layouts, styling, data fetching, errors, accessibility, and deployment-shaped app thinking.

## Git and GitHub

- About GitHub and Git: https://docs.github.com/en/get-started/start-your-journey/about-github-and-git
  - Use when the learner has not used version control or needs to commit the first demo.
- GitHub Skills: https://skills.github.com/
  - Use for hands-on GitHub practice when the learner needs interactive exercises.

## AI App Integration

- Vercel AI SDK Getting Started: https://ai-sdk.dev/docs/getting-started
  - Use for Next.js AI app integration.
- Vercel AI SDK Structured Data: https://ai-sdk.dev/docs/ai-sdk-core/generating-structured-data
  - Use when Week 4 needs stable JSON-like outputs.

## Claude

- Claude API Overview: https://platform.claude.com/docs/en/api/overview
  - Use for API prerequisites, keys, SDKs, and model access.
- Claude Tool Use: https://platform.claude.com/docs/en/agents-and-tools/tool-use/overview
  - Use only when the demo needs tool/function behavior.

## OpenAI Alternative

- OpenAI API Quickstart: https://developers.openai.com/api/docs/quickstart
  - Use when the learner chooses OpenAI or already has OpenAI setup.
- OpenAI Structured Outputs: https://developers.openai.com/api/docs/guides/structured-outputs
  - Use when the demo needs reliable schema-constrained output.
- OpenAI Function Calling: https://developers.openai.com/api/docs/guides/function-calling
  - Use when the demo needs model-triggered functions or tool calls.

## Data and Deployment

- Supabase Database Overview: https://supabase.com/docs/guides/database/overview
  - Use only when JSON/CSV is no longer enough.
- Vercel Deployments: https://vercel.com/docs/deployments
  - Use for publishing the demo.
- Vercel Environment Variables: https://vercel.com/docs/environment-variables
  - Use for API key and secret handling.

## Resource Selection Rules

- Assign 1-3 links per week, not a giant reading list.
- Pair every resource with a build task.
- For beginners, prioritize tutorials with immediate visible output.
- For intermediate learners, include structured output, tool use, testing, and deployment docs earlier.
- Do not assign algorithms, operating systems, distributed systems, or ML theory unless the user's target role requires it.
