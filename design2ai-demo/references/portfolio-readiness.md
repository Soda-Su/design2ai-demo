# Portfolio Readiness

Use this when the learner's goal is application, job search, Design Engineer transition, HCI/UX portfolio, or public project sharing.

## Portfolio Evidence

The final project should show:

- Problem: the real workflow pain or decision point
- User: who the demo helps
- Input: what the user gives the system
- AI role: what the model does and does not do
- Interface: how the user controls and reviews the AI output
- Stack: frontend, model API, data, deployment
- Testing: realistic tasks, failures, and fixes
- Reflection: what changed after testing

## Case Study Outline

```markdown
# Project Title

## One-Line Summary

## Context

## User and Problem

## Why AI Belongs Here

## Interaction Flow

## Technical Stack

## Data and Constraints

## Testing

## What I Would Improve Next

## Demo Link / Walkthrough
```

## Design-Background Advantage

Encourage the learner to foreground:

- Interaction clarity
- User journey
- Information architecture
- Visual hierarchy
- Scenario writing
- Evaluation criteria
- Before/after workflow comparison

These are not decoration. They are part of making an AI demo credible.

## Red Flags

Warn the learner if:

- The demo is only a wrapper around a prompt
- The UI hides the user's control
- The output cannot be tested
- The project claims too much career or technical mastery
- The case study cannot explain why AI improves the workflow
- There is no deployed link or walkthrough

## Final Package

Minimum:
- Live link or recorded walkthrough
- 5 screenshots
- Short case study
- Test notes
- GitHub repo if code quality is presentable

Better:
- Live link
- One-minute video
- Before/after workflow
- Testing table
- Stack diagram
- Short reflection on limitations
